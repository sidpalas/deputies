Use this file as a starting point for a real post, then set `draft: false` when it is ready to publish.

## Section Heading

Write posts in MDX. Markdown syntax works by default, and Astro components can be imported when a post needs richer
content.

## Images

Images in `apps/www/public` are served from the site root. For example, `apps/www/public/images/simple-mode.svg` can be
referenced as `/images/simple-mode.svg`:

![Simple Mode Deputies architecture](/images/simple-mode.svg)

Use HTML when you want a caption or more control over the markup:

  <img src="/images/integrated-mode.svg" alt="Integrated Mode Deputies architecture" />

</figure>

Use `astro:assets` for imported local images. Keep post-specific assets beside the post in an `assets/` directory so the
article and its images move together:

## Links And Callouts

You can link to [the Deputies repository](https://github.com/sidpalas/deputies) with normal Markdown.

> Use blockquotes for short notes, warnings, or quoted context from issues and docs.

## Embeds

Import `TwitterEmbed` when a post needs a full X/Twitter embed. The component fetches static oEmbed content at build time,
then progressively enhances it with X's widget script at runtime when available.

## Code

Use fenced code blocks with a language tag for syntax highlighting:

```js
const session = await deputies.sessions.create({
  prompt: 'Summarize the failing tests and propose a fix.',
});
```

Inline code like `mise run //apps/www:build` works too.

## Tables

| Feature             | Why it helps                                                           |
| ------------------- | ---------------------------------------------------------------------- |
| Frontmatter         | Keeps title, description, dates, and draft state validated.            |
| MDX                 | Allows Markdown content plus HTML or imported components when needed.  |
| Content collections | Powers the blog index, post routes, and RSS feed from the same source. |
