## Why this matters

`pnpm audit --prod` reports 14 advisories against the control plane: **1 critical** (`shell-quote` <= 1.8.3, command-injection class), **8 high** (`protobufjs` >= 8.0.0 < 8.0.2-class advisories, code-generation gadgets), and 5 moderate — all reachable only through `@daytona/sdk@^0.183.0`'s transitive dependency tree (its bundled OpenTelemetry OTLP exporters pull `protobufjs`; `shell-quote` comes in directly under `@daytona/sdk`). Daytona is the production sandbox provider for the deployed instance, so these sit on a real runtime path. Newer SDK releases exist (0.184.0–0.187.0 as of 2026-06-12), and patched versions of both vulnerable packages are published (`shell-quote@1.8.4`, `protobufjs@8.2.0+`).

## Steps

### Step 1: Bump `@daytona/sdk`

In `apps/control-plane/package.json`, change `"@daytona/sdk": "^0.183.0"` to `"@daytona/sdk": "^0.187.0"`. Run `npx pnpm@11.5.2 install`.

If install fails because `minimumReleaseAge` blocks 0.187.0, step down (0.186.0, 0.185.0, 0.184.0) until install succeeds, and note which version you landed on.

**Verify**: `npx pnpm@11.5.2 install` → exit 0; `git diff pnpm-lock.yaml | head` shows `@daytona/sdk` resolution changed.

### Step 2: Re-run the audit

**Verify**: `npx pnpm@11.5.2 audit --prod` → zero critical and zero high advisories. (Moderate advisories may remain; record them in your report but do not chase them.)

### Step 3 (only if step 2 still shows critical/high): Add targeted overrides

If the newest installable SDK still pins vulnerable transitives, add a pnpm overrides block to `pnpm-workspace.yaml`:

```yaml
overrides:
  shell-quote: '>=1.8.4'
  protobufjs: '>=8.2.0'
```

Re-run `npx pnpm@11.5.2 install`, then re-verify step 2. Add a one-line comment above the block: `# Security overrides for @daytona/sdk transitives; drop when the SDK ships patched deps.`

**Verify**: `npx pnpm@11.5.2 audit --prod` → zero critical/high.

### Step 4: Confirm the control plane still builds and the Daytona adapter still passes

**Verify**:

- `mise run //apps/control-plane:typecheck` → exit 0
- `mise run //apps/control-plane:test` → all pass (this includes `daytona-sandbox.test.ts`, which exercises the provider adapter against the SDK surface)
