# Packages

Reusable libraries shared by deployable apps belong here.

Examples that may fit here later:

- `config`: shared configuration helpers.
- `sdk`: client SDKs or typed API clients.
- `shared`: cross-app domain utilities and types.
- `types`: shared type-only contracts.
- `ui`: reusable UI primitives shared by web surfaces.

Deployable applications and services belong in `apps/`, not `packages/`.
